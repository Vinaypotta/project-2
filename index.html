Essential Git Commands Guide(cheat-sheet)

Short examples included for common workflows.

Quick checks

git --version - Show installed Git version
Use this to confirm Git is available on your machine.
$ git --version
git version 2.42.0

Repository Setup

git init - Initialize a new Git repository 
$ git init
Initialized empty Git repository in /path/to/repo/.git/
git clone [url] - Clone a repository from a remote source 
$ git clone https://github.com/user/repo.git
Cloning into 'repo'...

Basic Commands

git status - Check the status of your working directory 
$ git status
On branch main
Your branch is up to date with 'origin/main'.

Untracked files:
  (use "git add ..." to include in what will be committed)
    new-file.txt
git add [file] - Add a specific file to the staging area
Stage a single file before committing.
$ git add README.md
$ git status
Changes to be committed:
  (use "git restore --staged ..." to unstage)
    new file: README.md
git add . - Stage all changes in the current directory (new/modified files only)
Common when you want to stage all working directory changes in one step.
$ git add .
$ git status
Changes to be committed:
    modified: src/app.js
    new file: src/utils.js
git add -A - Stage all changes including deletions
Use this to ensure deleted files are staged as well.
$ git add -A
$ git status
Changes to be committed:
    deleted: old-file.txt
    modified: index.html
git commit -m "[message]" - Commit staged changes with a message 
$ git commit -m "Add initial project files"
[main abc1234] Add initial project files
 3 files changed, 120 insertions(+), 0 deletions(-)

Branch Operations

git branch - List all branches
git branch [branch-name] - Create a new branch 
$ git branch feature/login
$ git branch
  feature/login
* main
git checkout [branch-name] - Switch to a branch 
$ git checkout feature/login
Switched to branch 'feature/login'
git merge [branch] - Merge a branch into the active branch 
$ git checkout main
$ git merge feature/login
Updating abc1234..def5678
Fast-forward

Remote Repository

git remote add origin [url] - Add a remote repository
git push origin [branch] - Push changes to remote repository 
$ git push origin main
Enumerating objects: 10, done.
Counting objects: 100% (10/10), done.
git pull - Update local repository with remote changes 
$ git pull origin main
From https://github.com/user/repo
 * branch            main       -> FETCH_HEAD
Already up to date.

Tip: use git add -p to interactively stage hunks if you want finer control.
